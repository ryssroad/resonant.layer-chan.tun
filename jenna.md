# Мнение о Resonant Protocol

- Rust-часть собирается и `cargo test` проходит, базовые encode/decode, компрессия и шифрование протестированы — значит демо-сценарий «отправитель ↔ получатель» должен заработать из коробки.
- Python-примеры симметричны rust-примерам, пакет фреймов совпадает по формату, так что в пределах одного процесса/локальной сети обмен тоже будет работать.
- Флаги `ZSTD`/`XCHACHA` пока выставляются «для вида»: фактического сжатия/расшифровки нет ни в encode/decode, ни в примерах. Если подключить реальную компрессию/криптование, код придётся расширять.
- Заголовок хранит `slice_len`, но при разборе данные берутся из `shape`; несоответствие пропустится, а для `Q4` и `SparseCoo` расчёт размера сейчас условный. Это риск некорректных payload'ов.
- Флаг `STRONG_TAIL` и сама идея tail-хеша не доведены: вычисление есть, но отдельного контроля хвоста/stream-level интегритета нет.
- Handshake описан, но ответы и проверка совместимости пока чисто демонстрационные (в Rust получатель «напоминанием» печатает, что стоило бы ответить).

**Итого:** для MVP/дема — работает. Для продакшена потребуется довести обработку флагов, валидацию срезов, поддержку реальных dtype и протокол tail-хеша.
